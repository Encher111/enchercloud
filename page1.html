Lista komend oraz poradnik do języka EncherScript:

PORADNIK JAK ZACZĄĆ Z JĘZYKIEM ENCHERSCRIPT:
1. . Uruchom EncherScript IDE
2. Zaprogramuj program korzystając z komend zawartych w pliku
komendyencherscript.txt
3. Zapisz projekt, ważne że jeżeli ma być to pierwszy w kolejności
plik interpreterowany przez język musisz go nazwać "code"
4. Uruchom plik interpretera EncherScript który automatycznie
utworzy się w tym samym miejscu co zapisałeś projekt

KOMENDY JĘZYKA ENCHERSCRIPT:
Uwaga! Miejsca ze znakami "<", ">" należy zignorować, ten znak oznacza miejsce gdzie należy coś wpisać!

​

Uwaga! Znak >> oznacza przejście do kolejnej linii!

​

Uwaga! Znak *user* oznacza wpisanie czegoś przez użytkownika w uruchomionym programie i jednocześnie zatwierdzenie enterem

​

Uwaga! Przykład użycia oznacza zawartość pliku code.es

​

1. print - wyświetla na ekranie

przykład użycia: print Hello World >> end

efekt: Hello World

​

2. varprint - wyświetla zawartość zmiennej o podanej nazwie

przykład użycia: variable a 0 >> varprint a >> end

efekt: 0

​

3. variable - zmienia lub tworzy nową zmienną, i ustawia jej zawartość na

przykład użycia: variable a tekst >> varprint a >> end

efekt: tekst

​

4. input - tworzy nową zmienną o podanej nazwie i po uruchomieniu programu oczekuje na wpisanie czegoś, to co zostanie wpisane staje się zawartością zmiennej o podanej nazwie.

przykład użycia: input a >> varprint a >> end

efekt: *user*jakistekst* >> jakistekst

​

5. date - zapisuje aktualną date w zmiennej o podanej nazwie

przykład użycia: date a >> varprint a >> end

efekt: (załóżmy że dzisiejsza data to 10.07.2023) 10.07.2023

​

6. clock - zapisuje aktualną godzine w zmiennej o podanej nazwie

przykład użycia: clock a >> varprint a >> end

efekt: (załóżmy że obecna aktualna godzina to 18:06) 18:06

​

7. math - wykonuje operacje matematyczne takie jak dodawanie (+), odejmowanie (-), dzielenie (:), mnożenie (*) pomiędzy dwoma liczbami lub zmiennymi.

przykład użycia: math a + 2 c >> varprint c >> end

efekt: *user*8* >> 10

​

8. createfile <ścieżka do utworzenia pliku lub zmienna której zawartością jest ścieżka> - tworzy nowy plik w podanej ścieżce

przykład użycia: createfile C:\Users\barte\Desktop\e\plik.txt >> end

efekt: (utworzenie pliku plik.txt w lokalizacji C:\Users\barte\Desktop\e)

​

9. addtexttofile <ścieżka do dodania tekstu do pliku lub zmienna której zawartością jest ścieżka> - dodaje tekst do pliku o podanej ścieżce

przykład użycia: addtexttofile C:\Users\barte\Desktop\e\plik.txt jakistekst >> end

efekt: (dodano do pliku C:\Users\barte\Desktop\e\plik.txt tekst jakistekst)

​

10. addvartexttofile <ścieżka do dodania tekstu do pliku lub zmienna której zawartością jest ścieżka> - dodaje tekst ze zmiennej do pliku o podanej ścieżce

przykład użycia: input a >> addvartexttofile C:\Users\barte\Desktop\e\plik.txt a >> end

efekt: *user*fajnytekst* (dodano do pliku C:\Users\barte\Desktop\e\plik.txt tekst fajnytekst)

​

11. deletefile <ścieżka do pliku lub zmienna której zawartością jest ścieżka> - usuwa plik w podanej ścieżce

przykład użycia: deletefile C:\Users\barte\Desktop\e\plik.txt >> end

efekt: (usunięto plik plik.txt z lokalizacji C:\Users\barte\Desktop\e\plik.txt )

​

12. renamefile - zmienia nazwę pliku o podanej ścieżce na tekst

przykład użycia: renamefile C:\Users\barte\Desktop\e\plik.txt fajnyplik.txt >> end

efekt: (zmieniono nazwe pliku plik.txt na fajnyplik.txt w lokalizacji C:\Users\barte\Desktop\e\plik.txt(albo fajnyplik.txt zależy jak kto na to spojrzy))

​

13. copyfile <ścieżka do pliku lub zmienna której zawartością jest ścieżka> <ścieżka do pliku lub zmienna której zawartością jest ścieżka> - kopiuje plik o podanej ścieżce do podanej ścieżki

przykład użycia: copyfile C:\Users\barte\Desktop\e\plik.txt C:\Users\barte\Desktop\r >> end

efekt: (skopiowano plik plik.txt ze ścieżki C:\Users\barte\Desktop\e\plik.txt do ścieżki C:\Users\barte\Desktop\r

​

14. openurl - otwiera link do strony internetowej w domyślnej przeglądarce

przykład użycia: openurl www.google.com >> end

efekt: (otwarcie strony o adresie www.google.com w domyślnej przeglądarce)

​

15. openfile <ścieżka do pliku lub zmienna której zawartością jest ścieżka do pliku> - otwarcie pliku o podanej ścieżce

przykład użycia: openfile gra.exe >> end

efekt: (uruchomienie pliku gra.exe)

​

16. openfilealternative - działa identycznie jak openfile z tym że jest to stara metoda uruchamiania plików, nie zalecana, i nie zawsze działa

​

17. consolecolor - zmienia kolorki, wszystkie dostępne kolory można sprawdzić pod adresem: https://vrcc8d.webwave.dev

przykład użycia: consolecolor red yellow >> print Hello >> end

efekt: żółty napis na czerwonym tle "Hello"

​

18. wifilist - wyświetla listę dostępnych sieci wifi

​

19. systemcommand - wykonuje komendę systemową

przykład użycia: input a >> systemcommand a >> end

efekt: *user*echo Hello* Hello

​

20. poweroff - wyłacza komputer

przykład użycia: poweroff >> end

efekt: (wyłączenie komputera)

​

21. clear - wyczyszczenie ekranu

przykład użycia: clear >> end

efekt: (wyczyszczenie ekranu)

​

22. createfolder <ścieżka do utworzenia folderu lub zmienna której zawartością jest ścieżka do utworzenia folderu> - tworzy folder w podanej ściezce

przykład użycia: createfolder C:\Users\barte\Desktop\jakisfolder >> end

efekt: (utworzenie folderu jakisfolder w lokalizacji C:\Users\barte\Desktop\e)

​

23. deletefolder - usuwa folder w podanej ścieżce

przykład użycia: deletefolder C:\Users\barte\Desktop\e >> end

efekt: (usunięcie folderu e z lokalizacji C:\Users\barte\Desktop)

​

24. end - instrukcja wymagana na końcu każdego programu

przykład użycia: end

efekt: (nic się nie dzieje)

​

25. delay - odczekanie (s) - sekund (m) - minut (h) - godzin.

przykład użycia: delay 20 h >> end

efekt: (program będzie czekał 20 godzin)

​

27. ifvarpath - ustawienie nazwy pliku dla komendy ifvar

przykład użycia: ifvarpath code2.es

efekt: (ustawiono nazwę pliku dla komendy ifvar na code2.es)

ifvar - warunek - jeżeli zmienna o podanej nazwie będzie miała zawartość wpisaną w pole , to plik z którego interpretowany jest kod zmieni się z code.es na ten ustawiony w komendzie ifvarpath

efekt: (załóżmy że plik code2.es wyglada tak: print Ok >> end) Ok
